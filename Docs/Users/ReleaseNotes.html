WinMerge 0.2011.003.037 Release Notes

This experimental release of WinMerge 2011 allows for saving folder comparison results to an Excel spreadsheet.

The related commands can be invoked from the Folder Compare window's Tools menu:

  • Save to Excel spreadsheet
  • Show in Excel Viewer
  • Print with Excel Viewer
Note that the above commands apply to only the selected lines, so you may need to Select All from the Edit menu or hit Ctrl+A first.

If you face issues with some recent version of Excel Viewer, then Excel Viewer 2003 may be your friend.

Minor usability improvements:

  • The folder compare progress dialog now shows the paths of the files being compared. It was argued by some that, being lazily updated, this information is wrong most of the time. It was argued by others that they still want this information because it gives them a clue about files which require an unexpectedly large amount of processing time. I agree with the others.
  • Rearranging columns in Folder Compare window now preserves the current selection.
  • When navigating from a subfolder back to its parent, the selection within the parent folder will be set to the previously listed subfolder.

Jochen Neubeck, January 2013

WinMerge 0.2011.002.212 Release Notes

This experimental release of WinMerge 2011 makes the Compare As option accessible through the command line. The feature aims at supporting use cases like the one outlined here. In the given example, an extra parameter -t "script:\FileTransforms\xdoc2txt.wsc" is needed to tell WinMerge 2011 to pipe files through xdoc2txt. The reason is that WinMerge 2011 does not provide file transform automatisms like WinMerge 2 does (which is an explicit design decision driven by the realization that file transform automatisms have caused much confusion for WinMerge 2 users).

Furthermore, fixes are included to address issues #18 Binary compare reports false positives and #19 diff pane size not remembered.

Jochen Neubeck, May 2012

WinMerge 0.2011.002.145 Release Notes

This experimental release of WinMerge 2011 implements two substantial new features:
  • Line filters can now act as prediffers, i.e., operate on content before WinMerge passes it down to the diff algorithm. An example is given here. More examples are to follow.

    Prediffers can be specified in either of the following forms:

    • Capturing regular expressions

      All parts of a match which are covered by a submatch are removed, or replaced by a fixed sequence of characters, whose length is, for technical reasons, limited to the length of the shortest possible submatch.

    • Prediffer scriptlets

      A prediffer scriptlet implements a method with name ProcessLine, which is called once per line to perform preprocessing as desired, subject to the limitation that the length of the processed line must not exceed the length of the original line.

      By maintaining state, prediffer scriptlets are also capable of taking into consideration preceding content. For this purpose, they need to expose a method with name Reset to reset their state. WinMerge indicates the side on which a prediffer scriptlet is subsequently going to operate by passing the Reset method a value of 0 or 1. This allows a prediffer scriptlet to keep knowledge gathered from the left side for reference during processing of the right side.

  • The Open dialog's Include Subfolders option has been extended to support situations where folder structure differs between sides, but files on both sides are uniquely identifiable by their names, and hence can be matched without considering folder structure. When encountering name collisions, WinMerge attempts to resolve them by also considering sizes and modification dates of the involved files.

    To enable this special mode of operation, repeat clicking the Include Subfolders checkbox until it yields a visual appearance that is different from checked and unchecked, and an annotation shows up indicating that folder structure will be ignored.

Jochen Neubeck, April 2012
