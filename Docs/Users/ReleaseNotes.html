WinMerge 2011.003.523 Release Notes

This is a stable release of WinMerge 2011.
Significant new features since the preceding stable release include:

  • Line filters exist in a new flavor referred to as early-applied line filters:
    WinMerge.chm → Using Filters → Using line filters → Early-applied line filter syntax

  • The Open dialog's Include Subfolders option can be adjusted to ignore folder structure:
    WinMerge.chm → Quick start → Selecting files and folders to compare → 5.

  • Editor scriptlets are back:
    WinMerge.chm → Plugin scriptlets

  • Folder compare supports customizable background colors to make differences more visible.
    This feature can be configured through Edit → Options → Colors → List.

  • When comparing folders on machines with > 2 CPU cores, WinMerge creates multiple compare threads to get the job done faster.
    The number of compare threads is usually one less than the number of available CPU cores.
    A different value can be configured through Edit → Options → Compare → Folder → Number of compare threads.

  • The location bar remains responsive during drag-drop editing. This greatly eases drag-drop moving of text over long distances.
    While in drag-drop mode, you can stop scrolling by holding down the Shift key.

  • File compare allows for setting synchronization points to help in situations where WinMerge fails to reasonably match portions of involved files.
    After clicking into the lines you want synchronized, Merge → Synchronization Point will place a synchronization point at these lines.
    Repeat above steps to remove an existing synchronization point.
    Invoke Merge → Clear Synchronization Points to remove all synchronization points at once.
    This feature is adopted from Takashi's fork.

  • Unicode processing has been revised to support surrogate pairs rather than being actively destructive on them. A beneficial side effect of this enhancement is the option to choose between having combining characters visually combine with their preceding ones, or keeping them separate like WinMerge used to do due to its unawareness of the very concept.
    The option to Separate combining characters is accessible through Edit → Options → Editor.

  • Edit → Options → Colors now offers an option to Cross-hatch deleted lines.

  • Folder comparison results can be saved to an Excel spreadsheet.
    Besides the Save to Excel spreadsheet command, the Folder Compare window's Tools menu includes commands for viewing and printing with Excel Viewer. These assume that you use Excel Viewer 2003. Newer versions don't seem to allow for direct printing through command line, and also lack options to adjust column widths and apply auto filters. Be sure to also apply the updates located at Excel Viewer 2003 Service Pack 3 (SP3) and Security Update for Microsoft Office Excel Viewer 2003 (KB973484).

  • File/directory filters have been enhanced to allow for combining both inclusion and exclusion rules.
    Omitting the def: label in an .flt file makes f: and d: operate as inclusion rules unless prefixed with x, in which case they operate as exclusion rules.
    The f: and d: labels, again optionally prefixed with x to form an exclusion rule, can also be used with wildcard filters.

  • With LogParser 2.2 installed, you can enjoy more elaborate folder compare filter options by using SQL in your file/directory filters.
    All .flt files installed with WinMerge contain a default SQL filter which you can enable at your option when filling in the Open dialog. The default SQL filter allows you to specify minimum and maximum modification times and sizes. All fields are checked with IS NULL so as to have only the filled in fields take effect. SQL filters are enabled and filled in separately for each side. The default SQL filter can also be combined with simple wildcard filtering, in which case WinMerge loads it from the FileFilter.tmpl file.

  • Processing of archive formats which are unknown to 7-Zip can be delegated to external command line tools.
    An example is PowerBuilder library (.pbl) files. A command line tool to handle them can be found here.
    Read here on how to set up an ExternalArchiveFormat.ini to enable on-the-fly extraction of .pbl file contents.

  • Wrapper scripts can help automate tasks which would otherwise require user interaction.
    A first application of this new feature is GenerateHTMLReport.WinMerge, which you find in the WinMerge program folder.
    Invoking it without arguments provides usage information.

  • The setup now offers a Portable media option to install WinMerge in such way that it will not modify the registry of the system on which it is running.
    Known issue: On Windows XP and other pre-Vista systems, this option can only be used from an administrator account.

  • File → Start Collection opens a folder compare window in collect mode. WinMerge will prompt you for an existing reference folder to represent the right-hand side of the folder compare session, and create a temporary folder for the left-hand side.
    The reference folder will typically contain your working copy of some source code which is maintained in a source control system. Subsequent command line invocations will then add items to the existing folder compare window, rather than opening them to new file compare windows, provided they meet the condition of one of the specified files being located inside the reference folder, and the other one being located outside of it. The file that is located outside of the reference folder will be copied to the temporary left-hand side folder, assigning it the same name as the file located inside the reference folder, and creating subfolders as needed to match the structure of the reference folder.
    The feature aims at supporting offline code reviews, for which you would eventually store the collected items in an original vs. altered archive (a.k.a. Perry-style patch), using the context menu's Zip → Differences command.

  • ClearCase .mrgman files can be opened to a folder compare window, by dropping them onto the WinMerge main window.
    The header bars provide context menus to switch between base, source, and destination versions of the referenced files.

Bill Evjen's Blog gives advice in case WinMerge.chm fails to display content.
